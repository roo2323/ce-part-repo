# 테스트 계획 체크리스트

## 테스트 범위 정의
- [ ] 대상 모듈 목록
- [ ] 제외 범위 명시

## 테스트 유형 비율
- [ ] 단위 테스트: 60%
- [ ] 통합 테스트: 30%
- [ ] E2E 테스트: 10%

## 테스트 환경
- [ ] 테스트 DB (SQLite 또는 별도 PostgreSQL)
- [ ] Mock 서비스 목록
- [ ] 테스트 데이터 준비

## 일정
- [ ] 테스트 작성 기간
- [ ] 테스트 실행 기간
- [ ] 버그 수정 확인 기간

## 테스트 계획서 템플릿
```markdown
## [QA] 테스트 계획

**대상**: Phase {N} 완료 기능
**기간**: {시작일} ~ {종료일}

### 테스트 범위
| 모듈 | 우선순위 | 테스트 유형 |
|------|----------|------------|
| Auth | 🔴 높음 | 단위, 통합 |
| Contacts | 🟡 중간 | 단위, 통합 |
| Messages | 🟢 낮음 | 단위, 통합 |

### 테스트 피라미드
```
      /\
     /  \      E2E (10%) - 핵심 플로우
    /----\
   /      \    통합 (30%) - API 엔드포인트
  /--------\
 /          \  단위 (60%) - Service 함수
/____________\
```

### 환경
- DB: SQLite (테스트용)
- Mock: FCM, SendGrid

### 예상 시간
| 유형 | 예상 시간 |
|------|----------|
| 단위 테스트 작성 | 4시간 |
| 통합 테스트 작성 | 3시간 |
| E2E 테스트 작성 | 2시간 |
| 실행 및 버그 확인 | 1시간 |
```
