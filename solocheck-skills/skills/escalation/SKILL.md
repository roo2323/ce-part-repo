# 에스컬레이션 체크리스트

## 즉시 에스컬레이션 (🔴 Critical)
- [ ] 보안 취약점 발견
- [ ] 법적/윤리적 문제 발견
- [ ] 기술적 불가능 판단
- [ ] 일정 1일 이상 지연

## 판단 후 에스컬레이션 (🟡 Medium)
- [ ] SPEC 변경 필요
- [ ] 추가 리소스/비용 필요
- [ ] 에이전트 간 의견 충돌 (해결 불가)
- [ ] 외부 의존성 문제

## 에스컬레이션 불필요 (🟢 Low)
- [ ] 에이전트 내부 해결 가능
- [ ] 일정 영향 없음
- [ ] 기존 SPEC 범위 내

---

## 에스컬레이션 보고서 템플릿
```markdown
## [ORCHESTRATOR] 에스컬레이션 보고

**긴급도**: 🔴 Critical / 🟡 Medium
**유형**: {보안/법적/기술/일정/비용/충돌}

### 이슈 요약
{1-2문장으로 이슈 설명}

### 상세 내용
{발생 경위 및 상세 설명}

### 영향 분석
- 일정 영향: {있음/없음, 구체적 영향}
- 비용 영향: {있음/없음, 구체적 영향}
- 범위 영향: {있음/없음, 구체적 영향}

### 제안 옵션
1. {옵션 A}: {설명} - {장단점}
2. {옵션 B}: {설명} - {장단점}
3. {옵션 C}: {설명} - {장단점}

### 필요 결정
프로젝트 오너의 옵션 선택 필요합니다.
```
